01.系统概述

# 系统概述

本系统是 中芯超算（北京）科技有限公司 对外服务的超级计算集群。

# 硬件环境

硬件系统由计算阵列、服务阵列、存储子系统、互连通信子系统、监控诊断子系统等五大部分组成，下面针对主要部分进行简要介绍。

# 登录结点

  • 结点名称：目前提供 1个登陆结点，名称为 ln0

  • 主要功能：实现用户登录，程序开发，提交任务等工作

  • 禁止操作： 禁止用户再登陆节点运行计算任务

表1 登陆结点配置

指标 参数
CPU 型号 Intel Xeon Processor (Skylake)，核心数量 80 核
内存 128GB
硬盘 没有本地硬盘, 挂载共享存储
操作系统 版本 Centos 内核 4.18.0-240.el8.x86_64

# 计算结点

  • 结点名称：目前提供 19 个登陆结点，名称为 cn1 ~ cn19
  • 主要功能：实现用户的计算任务运算

表2 计算结点配置

指标 参数
CPU 型号 Intel Xeon Processor (Skylake)，核心数量 80 核
内存 128GB
硬盘 没有本地硬盘, 挂载共享存储
操作系统 版本 Centos 内核 4.18.0-240.el8.x86_64

# 互连系统

结点间的网络互联为 TCP/IP 网络互联协议。

# 存储系统

系统基于 NFS 网络文件系统搭建用户共享存储，对外提供统一存储分区，供所有登陆结点与计算结点进行挂载。

作用：

NFS高性能存储用于用户进行计算任务的数据存储，用户在登陆系统后看到的数据存储在 NFS存储中。

常用目录说明：

  • 账号根目录：/work/home/$username 该目录属于共享目录的下级目录，存放账号根目录。例如账号 zhang3 在存储系统的账号根目录为 /work/home/zhang3。

  • 常用软件安装目录：/work/software 该目录属于共享目录的下级目录，存放用户常用的软件或编译器等。该目录仅提供用户普遍需要使用的常用软件，会随着用户的需求不断更新，软件由管理员负责安装，为用户提供使用。

注意：为了满足并行计算对共享存储的高速读写需求，NFS网络存储系统 不提供多副本服务，因此无法长期保证用户数据的安全，建议用户及时拷贝自己的核心数据结果。

# 加载方式

# 加载方式1：使用 export 命令

用户可以使用 export 命令进行软件环境设置，进而加载系统预编译的应用程序。

例如，用户希望使用 wrf 软件，就可以前往 /work/software 目录寻找 wrf 软件，如果发现该软件已经安装，则可以通过 export 命令配置环境变量，实现软件的加载。

# 加载intel编译器
source /work/software/intel/2018u4/compilers_and_libraries/linux/bin/compilervars.sh  intel64
# 加载mpi动态库
export LD_LIBRARY_PATH=/work/software/intel/2018u4/compilers_and_libraries/linux/mpi/intel64/lib:$LD_LIBRARY_PATH
# 加载可执行程序
export PATH=/work/software/wrf/4.2.2-icc18-impi/main:$PATH

# 加载方式2：使用module 命令

系统采用 Environment modules 工具用来快速设置和修改用户编译运行环境。Environment modules 通过加载和卸载 modulefile（环境配置文件）可直接改变用户的环境变量，用户不需要频繁修改 ~.bashrc 文件，从而避免误操作。

如果用户计划使用某一软件，常用步骤为：

  1. 先在 shell 命令行下输入 module av 查看所需软件是否已在系统中安装；
  2. 若已安装，则通过 module help 命令具体查看所需软件的版本、依赖环境等；
  3. 然后在 shell命令行下通过 module load 加载环境变量；
  4. 最后提交脚本。

如下为 Modules 常用命令：

表 3 Modules 常用命令

命令 作用 示例
module avail | av 查看系统中所有可用软件 module av
module avail | av 模块名 查看系统指定名称开头的软件 module av intel
module add | load 加载环境变量 module load intel/2018u4
module rm |unload 卸载环境变量 module unload intel/2018u4
module list | li 显示用户已加载的环境变量 module li
module purge 卸载当前 shell 环境下的所有环 module purge

例如，用户希望使用 wrf 软件，可以使用 module av wrf 进行查看，然后执行 module load wrf/4.2.2-icc18-impi 命令即可使用了。

# 编译环境

# Intel编译器

系统上已安装 Intel Compiler 2018 update4 版本，建议加载方式为：

module add intel/2018u4
1

# GCC编译器

系统默认安装的 GCC 版本是 8.4.1，相关的编译命令都安装到 /usr/bin 目录中，可以直接使用。

# MPI编译器

系统上已安装 Intel IMPI 2018 update 4 版本，建议加载方式为：

module add impi/2018u4
1

使用 MPI 编译器编译程序，建议设置相关环境变量，常用的有：

表4 GCC 编译器 + IMPI 编译器

环境变量 语言 GCC编译器 GCC对应的MPI编译器 Intel 编译器 Intel编译器对应的MPI编译器

F77 / F90 / FC Fortran gfortran mpif77 ifort mpiifort

配置举例

如果用户使用 GCC + IMPI 进行编译，可以设置如下环境变量：

# 串行程序
export CC=gcc CXX=g++ F77=gfortran F90=gfortran FC=gfortran
# 并行程序
export CC=mpicc CXX=mpiicxx F77=mpif77 F90=mpif90 FC=mpif90
