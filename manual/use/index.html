02.使用方式

# 使用方式

为了更好的保证用户的数据安全，在用户使用系统资源前，需要具备如下条件：

# 基本条件

用户需要具备的基本条件如下：

  1. 经过用户基本审查创建流程，并填写了相应的文件和协议。
  2. 具备一个系统用户及密码（或密钥文件）。

具备上了上述条件，您就可以尝试登陆使用系统资源，登陆 步骤及所需软件如下节描述。

# 远程登陆

使用 ssh 客户端软件（如 SSH Secure Shell Client，SecureCRT，Putty）进行远程链接。

链接配置为：

  • 远程主机 Remote Host : 111.198.60.163
  • 端口号 Port：可以输入 53103 / 53104 / 53105 / 53106 其中之一，例如 53103。
  • 账号名 UserName: 请输入系统账号名。
  • 验证方式（二者选其一）：
    • 密码 PassWord：请输入系统账号密码。
    • 证书文件 User private key：请上传系统账号对应的证书文件。

Tips： SSH Secure Shell Client，SecureCRT，Putty 等均为免费软件，其他可选常用 ssh 工具还有 Xshell ，Mobaxterm，网络上均有下载。

使用 命令行方式 进行远程链接。

打开终端，使用 ssh 命令进行链接，例如：

# 命令格式1
ssh user@host -p port
# 举例
ssh zhang3@111.198.60.163:5313

# 文件传输

使用 sftp 客户端 进行文件传输。

常用客户端软件：

  • SSH Secure Shell Client ：本身自带的文件传输功能
  • WinScp : 免费软件，网络容易下载，且该软件支持断点续传

使用以上软件即可实现文件传输功能，推荐使用 WinSCP 传输软件。

警告！系统中的存储空间只作为数据的临时存储，请用户及时把重要数据或敏感数据保存到自己的计算机中，并及时清理自己的存储空间。

使用 命令行方式 进行进行文件传输。

打开终端，使用 scp 命令进行数据传输，例如：

# 上传传文件
scp -P 端口号 本地文件 用户名@远程主机:远程目录
# 下载文件
scp -P 端口号 用户名@远程主机:远程文件 本地目录

# 环境变量设置

根据用户帐号使用的 Shell 的不同，设置环境变量的方法也有所不同。假设我们要增

加一个用来表示字符串 /usr/local/bin 的环境变量 MYENV，可以采用下面的方法来设置。

系统默认用户选择的环境变量为 Bash。

  1. Bash 的设置方法
export MYENV=/usr/local/bin
1

2.sh 的设置方法

MYENV=/usr/local/bin 
export MYENV
1
2

Tips： 如果需要环境变量在登录进用户帐号后自动设置，则可以编辑用户帐号目录下的 ~.bashrc 文件，将上述命令行加入文件中。

# 退出系统

执行 exit 命令或按 ctrl-d键，即可退出系统。

# 用户账号密码修改

用户可以通过 passwd 命令修改用户密码，以 zhang3用户为例，举例说明如下：

[zhang3@ln0 ~]$ passwd
Changing password for user zhang3.
Current password:
New password:
Retype new password:
passwd: all authentication tokens updated successfully.
1
2
3
4
5
6

修改步骤：

  1. 首先需要输入账户的原始密码，
  2. 之后再输入新的密码，重复输入一次后，就会显示密码更新成功。

Tips：

  1. 输入新密码时会有密码强度校验，请输入包含大写字母、小写字母、数字和特殊字符的密码，最小长度为 8 位。
  2. 输入密码过程中 不会 显示已输入的字符串！

特别说明：为了保证您用户的数据安全，我们采用了多种方法和技术手段，但您也需要保证您的系统用户密码不外泄。希望您能定期主动更改登陆密码（两个月更换一次为宜）。
